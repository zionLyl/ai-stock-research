# 投资逻辑追踪

每只持仓都需要一个书面逻辑。说不清为什么买 = 不该买。

## 数据模型

每只股票的逻辑存储为 `memory/thesis_{CODE}.md`:

```markdown
# {NAME}({CODE}) 投资逻辑

## 基本信息
- 创建: YYYY-MM-DD
- 板块: XX
- 市值: XX亿
- 仓位: XX万

## 看多逻辑
1. 核心理由1（数据支撑）
2. 核心理由2（事件催化）

## 风险
1. 最大风险（影响评估）
2. 次要风险

## 催化剂
- [ ] 催化剂1 — 预计时间
- [ ] 催化剂2 — 预计时间

## 止损条件
- 价格止损: XX元 (-X%)
- 逻辑止损: 如果XXX发生则卖出

## 确信度: X/5

## 更新日志
- YYYY-MM-DD: 创建
```

## 工作流

### 创建新逻辑
1. 获取基本面数据 (`cn_data.py`)
2. MCP 搜索近期新闻/催化剂
3. 填充上述模板
4. 设定止损位 (config_manager)

### 更新逻辑
触发条件：财报发布、重大新闻、价格大幅波动
1. 重新获取数据
2. 对照原逻辑检查是否有变化
3. 更新确信度
4. 记录变化到更新日志

### 清理逻辑
卖出后：归档到 `memory/thesis_{CODE}_closed.md`，记录收益和教训。
